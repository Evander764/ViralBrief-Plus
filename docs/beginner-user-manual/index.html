Viral Brief Plus 本地使用说明

给完全不懂代码的人看的使用说明书

这份说明只讲“怎么用”。你可以把 Viral Brief Plus 理解成一个本地内容雷达：它会帮你巡检小红书和抖音账号，保存候选内容，筛出达标内容，再生成每日爆款选题日报。

打开就能看

本说明书是单个 HTML 文件。双击本文件夹里的 index.html 即可阅读。

不改代码

日常只需要点按钮、填账号、确认数字、导出日报。

数据在本机

内容库、截图、导出文件和 API Key 都保存在你自己的电脑里。

1. 这个应用是做什么的

Viral Brief Plus 是一个“本地内容情报工作台”。它不是帮你刷平台，也不是直接替你决定选题，而是把内容采集、筛选、复核、日报整理这几步放在一个本地工具里。

自动巡检账号

按你的账号池，先巡检小红书，再巡检抖音，打开内容详情并保存截图和数据。

只把靠谱内容入榜

数字、时间、是否达标都由程序判断；看不清的内容会留给你确认。

生成每日选题日报

达标内容进入日报，AI 负责整理母题、原因、标题建议和承接建议。

最重要的一句话：AI 不负责判断“有没有达标”。点赞、收藏、转发、发布时间和入选规则，都由本地程序按固定规则处理。AI 只负责把已经达标的内容整理成更好读的日报。

2. 使用前准备

开始前先确认下面几件事。准备好以后，后续每天基本就是打开、巡检、确认、生成日报。

  • 电脑上已经可以打开 Viral Brief Plus。
  • 电脑上安装了 Chrome 浏览器。
  • Chrome 里已经登录小红书和抖音。
  • 如果平台出现验证码、登录页或风控提示，先在 Chrome 里手动处理。
  • 你有一批想长期观察的博主账号。
  • 如果要生成 AI 日报，需要准备一个可用的 AI API Key。

如果你只想先建账号池、采集内容、人工看内容，可以先不填 API Key。真正要生成带 AI 分析的日报时，再去“设置”里填 Key。

3. 第一次打开应用

打开 Viral Brief Plus

如果你拿到的是 Mac 应用，双击 Viral Brief Plus.app。如果你拿到的是项目文件夹版本，让安装的人先帮你启动应用。

看到本地工作台

正常情况下会出现一个本地窗口，或浏览器打开 http://127.0.0.1:8787。这个地址只代表你的电脑本机，不是公开网站。

认识顶部五个页面

顶部有“概览、内容库、账号池、每日结果、设置”。日常主要用“概览、账号池、每日结果”。

概览

看总数、启动自动巡检、粘贴链接抓取、处理今日候选。

内容库

查看历史采集内容，按状态和平台筛选。

账号池

添加、发现、编辑、打开和管理要巡检的博主账号。

每日结果

生成、查看、导出或删除日报。

设置

保存 API Key、设置自动运行时间、调整巡检标签数、清理缓存。

4. 第一次设置 API Key

API Key 可以理解成“调用 AI 的钥匙”。它用于生成日报里的总结、选题和标题建议。应用会把 Key 加密保存在你的电脑上。

进入“设置”页面

找到“AI API Key（BYOK）”区域。BYOK 的意思是“使用你自己的 Key”。

选择供应商

供应商可以选 OpenAI、DeepSeek、小米 MiMo、Kimi、智谱、通义千问、硅基流动、Anthropic Claude 或自定义。选好后，接口地址会自动填入。

填写模型名

选择供应商后，模型输入框会给出示例。不会选时，先用系统提示的默认或示例模型。

粘贴主 Key 并保存

把 Key 粘贴到“主 API Key”，点击“保存主 Key”。应用会保存并自动测试。保存后输入框会清空，这是正常的。

可选：保存备用 Key

如果你有第二个 Key，可以放到“备用 API Key”。主 Key 出问题时，后续更容易切换。

你看到的提示 代表什么 该怎么做
Key 可用 供应商、模型和接口地址能正常调用。 可以继续设置账号池和生成日报。
已保存，但测试失败 Key 已经存进本机，但网络、模型名、额度或接口地址可能有问题。 先检查模型名和供应商，再确认 Key 是否有余额或权限。
没有 API Key 可以采集内容，但 AI 日报无法完整生成。 去对应平台申请 Key 后再填。

5. 建立账号池

账号池就是你想长期盯的博主名单。正式日报只收账号池里的小红书和抖音内容，所以账号池越准确，日报越稳定。

三种添加方式

方式 适合什么时候用 怎么操作
发现关注账号 你已经在 Chrome 登录平台，并且关注了很多博主。 进入“账号池”，点击“发现关注账号”，等它同步到本地账号池。
手动填写 你已经知道博主昵称、平台和主页链接。 选择平台，填写昵称、分类、优先级、主页链接，点击“添加到账号池”。
搜索填空 你只知道名字，不确定主页链接。 输入昵称，点击“打开平台搜索”，在 Chrome 找到本人主页，复制主页链接回来保存。

账号池里每一列怎么理解

字段 解释 建议
巡检 勾上后，这个账号会参与自动巡检。 暂时不想看某个账号，就取消勾选。
平台 小红书、抖音或其他。 正式日报主要看小红书和抖音。
优先级 S、A、B 代表先后顺序。 最重要账号设 S，其次 A，普通观察设 B。
主页链接 应用巡检时会打开这个链接。 尽量填写真实主页链接，不要填单条内容链接。
最近巡检 这个账号上一次实际跑完巡检的日期。 当天已经跑过的账号默认不会重复跑。

“打开全部小红书主页”适合人工检查账号是否正确。应用会把有效的小红书主页一次性打开到 Chrome 多个标签页。

6. 每天怎么用

下面是一套最稳的日常流程。第一次会稍慢，账号池建好以后会轻很多。

先确认 Chrome 登录正常

打开 Chrome，确认小红书和抖音都能正常访问。如果出现登录、验证或空白页，先手动处理。

打开 Viral Brief Plus

进入“概览”。如果顶部显示没有 Key，而你今天要生成 AI 日报，先去“设置”保存 Key。

点击“一键自动巡检”

应用会先跑小红书阶段，再跑抖音阶段。默认每轮同时打开 6 个账号标签页，账号跑完会关闭自己的标签。

等待巡检完成

巡检期间不要频繁手动操作正在被应用控制的 Chrome 标签页。需要中断时，点击“停止巡检”。

处理“今日候选池”

检查标题、作者、发布时间、点赞、收藏、转发、评论、账号池关联。确认无误后点击“确认入库”。

生成日报

去“每日结果”点击“生成日报”，或在“概览”选择生成。生成后可以查看日报和导出文件。

巡检阶段会尽量“先保存，再筛选”。所以你可能会看到未达标、缺时间、缺数字的内容出现在内容库里，这是正常的。它们只是被保存下来，不代表已经进入日报。

7. 怎么确认候选内容

候选池是人工复核区。只要数据来源不够强，应用就会把内容留给你确认，避免把不靠谱数字写进正式日报。

确认一条内容时看什么

  • 截图或封面是不是这条内容。
  • 标题、作者是否正确。
  • 发布时间是否填对。
  • 小红书是否有点赞和收藏。
  • 抖音是否有点赞、收藏和转发。
  • 账号池是否关联到正确博主。
  • 内容类型是视频还是图文/文章。
  • 明显不需要的内容可以归档或删除。

状态标签是什么意思

状态 含义 会不会进日报
已确认 人工或稳定来源确认过，且会重新计算是否达标。 只有已确认且达标的内容才可能进入日报。
待复核 数据来自弱来源、截图识别或普通页面文本，需要你看一眼。 不会，确认后才有机会进入。
发酵中 还没确认，可能还在观察期。 不会，确认后再判断。
缺点赞/收藏/转发数 关键数字缺失。 不会，需要补齐并确认。
未达阈值 已知数字低于入选线。 不会，但仍会保存在内容库。
重复/已归档 重复内容或你主动归档的内容。 不会。

入选规则，用人话说

小红书

点赞超过 1000，并且收藏超过 1000，才算互动达标。

抖音

点赞超过 1000、收藏超过 1000、转发超过 1000，三项都要达标。

账号池

内容必须关联到账号池里的小红书或抖音账号。

注意：空白数字不等于 0。空白代表“还不知道”，需要复核。平台显示 1000+ 算超过 1000；如果正好是 1000，不算达标。

8. 生成、查看和导出日报

进入“每日结果”

点击“生成日报”。如果今天还没有达标内容，日报可能显示达标 0 条，这是正常结果。

查看日报

生成后点击“查看日报”，可以在应用里预览本地 HTML 日报。

导出文件

点击“显示 Markdown”“显示 CSV”或“显示压缩包”，应用会在 Finder 里显示对应文件。

复用日报

Markdown 适合复制到文档或飞书，HTML 适合直接浏览，CSV 适合表格整理，压缩包适合打包归档。

导出类型 适合用途
HTML 打开后直接阅读，或用浏览器打印成 PDF。
Markdown 复制到文档、知识库、公众号草稿前的整理区。
CSV 用表格查看入选内容、指标和链接。
ZIP 压缩包 把日报相关文件一次性打包给别人或留档。

9. 自动运行和安全瘦身

每日自动生成

在“设置”里打开“开启每日自动生成”，选择每天运行时间和默认回溯天数。适合每天固定时间做内容雷达。

自动运行依赖本地应用服务和 Chrome 登录状态。电脑休眠、应用没开、平台要求重新登录时，自动巡检可能无法完成。

浏览器巡检标签数

默认每轮同时打开 6 个账号标签。电脑比较吃力时，可以在“设置”里把“每轮同时打开账号标签数”调低，例如 2 或 3。最多建议不要超过 10。

本地数据安全瘦身

“设置”里的“本地数据”区域可以查看数据库、截图和 RPA 浏览器缓存占用。点击“安全瘦身”会清理可安全删除的缓存，保留登录态、数据库、API Key 和近期截图。

10. 数据和隐私说明

本机服务

应用默认只在你电脑自己的地址 127.0.0.1 上运行，不是公开网站。

Key 加密保存

API Key 保存在桌面端，本地加密；浏览器插件不保存 Key，也不调用模型。

导出在本地

日报、截图和数据库都在本地目录里。你可以自己决定是否分享导出文件。

文件一般在哪里

你用的版本 数据位置 说明
Mac 应用版 ~/Library/Application Support/Viral Brief Plus 数据库、配置、截图和导出文件通常保存在这里。
项目文件夹版 data/ 数据库在 data/viral-brief-plus.db，导出在 data/exports/，截图在 data/screenshots/。

11. 常见问题

生成日报是空的，为什么？

日报只收“账号池内、已确认、平台正确、发布时间在窗口内、关键指标达标”的内容。最常见原因是还没确认、没有关联账号池、缺关键数字，或者数字没超过 1000。

巡检时 Chrome 打开了很多标签页，正常吗？

正常。应用默认每轮同时打开 6 个账号标签页，账号跑完会关闭自己的标签。如果电脑卡，可以去“设置”把标签数调低。

巡检卡住、平台出现登录页或验证码怎么办？

先点击“停止巡检”。然后到 Chrome 里手动登录、验证或刷新平台页面。确认平台可正常浏览后，再回到应用重新巡检。

提示已经有巡检在运行，不能再启动怎么办？

同一时间只允许一个巡检。等待当前巡检完成，或点击“停止巡检”。这是为了避免两个巡检同时控制 Chrome，导致数据混乱。

粘贴链接抓取没有抓到数字，是不是坏了？

不一定。平台经常只返回空壳页面、验证页或隐藏指标。应用会尽力抓标题和页面信息，抓不到的数字会留给你手动补录。

为什么内容库里有未达标内容？

应用的原则是“先保存，再筛选”。未达标内容保存在内容库，方便以后复查，但不会进入正式日报。

为什么我确认了内容，还是没进日报？

确认只是第一步。它还必须关联账号池，发布时间在回溯窗口内，并且小红书点赞和收藏都超过 1000，或抖音点赞、收藏、转发都超过 1000。

API Key 测试失败怎么办？

检查四件事：供应商选对了吗，模型名填对了吗，Key 有没有额度或权限，当前网络能不能访问该供应商。应用提示“已保存但测试失败”时，Key 仍然已经保存在本机。

应用提示 Node.js 版本过低怎么办？

把提示截图给安装的人处理。Viral Brief Plus 需要 Node.js 22.5 或更高版本。日常使用者不需要自己理解这是什么。

插件配对 token 是什么？

它相当于浏览器插件和本地应用之间的配对码。只有插件要把网页内容保存到桌面端时才需要。重置 token 后，需要同步更新插件设置。

12. 术语翻译

术语 普通说法
账号池 你要长期观察的博主名单。
巡检 应用按名单去平台看最新内容，并把可采集内容保存下来。
候选池 今天新发现、需要你确认或补录的内容。
确认入库 你检查数字和信息无误后，允许它参与正式筛选。
回溯天数 日报要看最近几天的内容，比如最近 1 天、3 天或 7 天。
BYOK 使用你自己的 AI API Key。
阈值 入选门槛。这里主要指互动数要超过 1000。

13. 一页清单

忘了步骤时，只看这一页也可以。

第一次使用

打开应用，进入“设置”保存 API Key；进入“账号池”添加或发现账号；确认 Chrome 已登录小红书和抖音。

每天巡检

进入“概览”，点击“一键自动巡检”，等小红书和抖音阶段跑完。

确认候选

在“今日候选池”补齐数字、发布时间和账号池关联；准确的点“确认入库”，不需要的点“归档”。

生成日报

进入“每日结果”，点击“生成日报”，然后查看日报或显示导出文件。

日报为空时

回到候选池检查：是否已确认、是否关联账号池、是否超过 1000、是否在回溯日期内。
